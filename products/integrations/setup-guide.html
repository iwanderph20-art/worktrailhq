Setup & Integration Guide

Welcome! Worktrail is one file (worktrail.html). Nothing to install, no monthly fee. This guide gets you running so employees can clock in from their own phones — all synced through one link.

Part 1 — Get it running (about 5 minutes)

Worktrail runs synced across every device using a free Google Firebase database that you own — we never see or store your data. This is the standard setup.

STEP 1

Create your free Firebase database (~2 min)

  1. Go to console.firebase.google.com and sign in with any Google account.
  2. Click Add project → give it any name → continue (skip Google Analytics).
  3. Left menu: Build → Realtime Database → Create Database.
  4. Pick a location → choose Start in test mode → Enable.
  5. Copy the URL shown at the top — it looks like
    https://your-project-default-rtdb.firebaseio.com
STEP 2

Set up Worktrail

  1. Open worktrail.html (host it first — see Part 4).
  2. Click Set Up My Kiosk: business name/logo/color → admin password → paste your Firebase URL on the last step → Finish.
  3. Set your database security rules (Part 2).
  4. Add employees (Admin → Employees) and give each a 4-digit PIN.
STEP 3

Share ONE link with all staff

In Admin → Settings → Kiosk Link → Share with Employees, copy the generated link and send it to everyone. Whoever opens that link — on any phone or tablet — is automatically connected to the same synced data. One link, everyone synced, no accounts for employees.

💡 Single-device only? During setup you can choose “Use on a single device only,” but then records stay on that one device and staff can’t clock in from their own phones. We recommend the synced setup above for almost everyone.

Part 2 — Lock down your database (do this once)

Your records live in your Firebase, so you control access. In the Firebase console → Realtime Database → Rules, replace the rules and click Publish.

Simple (shared kiosk — anyone with the link can read/write; fine for most small teams):

{ "rules": { ".read": true, ".write": true } }

Tighter (only under the Worktrail path):

{
  "rules": {
    "sd": { ".read": true, ".write": true },
    "$other": { ".read": false, ".write": false }
  }
}

Unsure? The simple rule is safe to start with. Email support@worktrailhq.com and we’ll help you tighten it.

Part 3 — Connect to your current software

You don’t replace your payroll or HR system — Worktrail feeds it. Three ways, easiest first:

1. Export a file (no setup)

Admin → Records → CSV or PDF, or Payroll → Download PDF. Import into QuickBooks, Gusto, ADP, Excel, or Google Sheets.

2. Live webhook (no code — recommended)

Admin → Settings → Webhook / Integrations → paste a URL → Save & Send Test Event. Every clock-in/out is sent as JSON in real time:

  • Google Sheets — paste-ready script included (google-sheets-webhook.gs).
  • Zapier / Make — “Catch Hook” / “Custom webhook” → 6,000+ apps (Slack, SMS, payroll, CRM).
  • Your own system — any HTTPS endpoint you control.

See the Webhook Guide for step-by-step.

3. Read directly from your Firebase (developers)

Your clock data sits under the sd/timesheet path in your Firebase. Your own scripts, a BI tool, or Zapier’s Firebase integration can read it directly.

Worktrail runs in the browser, so webhooks/exports happen while the kiosk is open — exactly when clock-ins occur. Need always-on, enterprise-grade sync or a custom integration? Email support@worktrailhq.com for a quote.

Part 4 — Put your kiosk online (so the link works everywhere)

For staff to open the same link on their own phones, the worktrail.html file must live at a public web address. Pick any free host (2 minutes, no coding):

  • Netlify Drop — go to app.netlify.com/drop and drag worktrail.html in. Instant public URL. Easiest.
  • Cloudflare Pages / GitHub Pages — free static hosting.
  • Your existing website — upload it to your own domain (e.g. yourbusiness.com/kiosk.html).

Once it’s online, redo Settings → Kiosk Link → Share with Employees so the link uses your public address, then send it to staff.

⚠️ Opening the file by double-click (file://…) works only on that one computer, and phone cameras stay off. A public https:// address makes the shared link and photo capture work on every device.
